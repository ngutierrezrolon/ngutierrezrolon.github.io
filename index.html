Easy Sentiment Analysis

This is a short project that attempts a sentiment analysis problem without Keras or other ML. The dataset includes 10,000 restaurant reviews, which are then pre-processed, cleaned and tokenized so that then using a downloaded pre-tagged lexicon, a calculating function can be created that counts the number of positive, neutral and negative word instances and finally adds up to a numerical result. This is then compared with the 1-5 rating given by the customer.

View Code

the graph shows how the calculated scores match against the empirical rating by the customer. Notably, only the reviews which had been given scores of 1 and 2 present negative calculated scores (with a few outlier exceptions), while all reviews equal or above 3 received positive calculated scores.

Pokemon data analysis

This is a short project using techniques of data exploration, visualisation to investigate, using statistically rigorous techniques, some of the variables and their correlations.

View Code

The previous graph shows the density distributions of the main variables.

The analysis features some statistical tests for correlation. In one instance, the Welch two sample test is used to compare the average power statistics for legendary and non-legendaries. Additionally, after performing a simple Anova test to check for correlations between weight and both type variables, a post-hoc Tukey's HSD test is performed on each of the type pairs to take notice of how type pairs interact and find which type pairs on average have the highest and lowest weight. This and more statistical analysis of pokemon data on github.

Blog Entry 1: Making a website

Making a website is very useful and can be very simple. This website is my personal portfolio and was made using HTML, CSS and Javascript.

HTML

HTML (HyperText Markup Language) is the part responsible for the structure of the content in the webpage. The subsections, headers, footers, hyperlinks, and main body of text are all done in this language.

CSS

CSS (Cascading Style Sheets) is the part responsible of all the main visuals, including fonts colors (in hexcode), spacing, as well as making the margins dynamic to accomodate mobile viewing.

JavaScript

Javascript is the most dynamic part, used to add interactivity, constantly updating sections, or any other advanced feature. The animation in projects section is all done in javascript.
Making your own website is relatively simple and very satisfying work.

Blog Entry 2: What is a "good enough" shuffle?

Random shuffles, randomness, card shuffling algorithms and more. If you like a good game of cards, whether that be (game1), game2 or game3, most card games start with a shuffle. As the game starts and the people around the table look at the cards, they groan and complain “who shuffled these?”. I want to talk about this conundrum of the shuffle, what the meaning of the word “random” even is, and explore how its practically applied and computationally replicated. There's several standard ways to shuffle.

1- The Overhand Shuffle

Not all shuffles are mixing the same amount of cards, if you think about it. The basic overhand shuffle is removing a sector of cards from the deck, merging the rest of the deck and then adding them back on top. In this case a sector of size n is simply being displaced, but that sector will retain its original order entirely.

2- The Perfect Shuffle

Most people would describe a shuffle as breaking the deck in two halves and then intermixing them. If the interleaving of the cards is done such that the two packets strictly alternate, then this is known as a perfect shuffle. Other names for it are Faro shuffle in USA, or Weave shuffle in UK. Depending on where the original top and bottom cards go, they are considered an out-shuffle or an in-shuffle. In an out-shuffle, the first and last card retain their positions exactly. In an in-shuffle, the first card will now go second, and the last card will go second-to last. k perfect in-shuffles will restore the order of an n-card deck if . 3- The Riffle

Most people don't perform perfect shuffles, and doing these would actually be to the detriment of randomness desired for card games. This shuffle isn't random, it produces an entirely predictable pattern. This is also why Faro shuffles are common in magic tricks. What most people do is normally referred to as a Riffle Shuffle. the deck is partitioned in two, then interleaved arbitrarily. For example, groups of sizes between 1-4 are interleaved from each half of the deck. A mathematically precise definition exists credited to Gilbert-Shannon-Reeds. The Gilbert-Shannon-Reeds model assigns probabilities to each permutation. This mathematical result suggests 7 riffle shuffles to produce a sufficiently random pattern, such that this combination might not have been seen ever before.
